Coordinate Frames

Coordinate frames

  • A point is described by a coordinate vector.

    • The vector represents the displacement of a point w.r.t. some reference frame ‚Äì we call this a bound vector.
    • Notice that a point is not the same as a vector!

  • A coordinate frame, is a triad of orthogonal axes which intersect at a point, known as the origin.

  • A real object comprises infinitely many points.

    • An object, unlike a point, has an orientation.
    • Attaching a coordinate frame to an object, every point within it can be described by a constant vector w.r.t. that frame.

  • This allows us to describe the position and orientation, the pose, of that object‚Äôs fixed frame w.r.t. a reference coordinate frame.

Coordinate frames


  • We think of AŒæB{^A}\xi_B as describing some motion: pick up {A}\{A\}
    • Apply a displacement and a rotation,
    • Now {A}\{A\} is transformed to {B}\{B\}.
  • If the initial superscript is missing, we assume that the pose is w.r.t. the world coordinate frame.
  • The point ùêè\boldsymbol{P} can be described w.r.t. either coordinate frame: Aùê©=AŒæB‚Ä¢Bùê©. {^A}\bm{p} = {^A}\xi_B \bullet {^B}\bm{p}.
    • The operator ‚Ä¢\bullet transforms the vector.
    • The new vector describes the same point but w.r.t. a different coordinate frame.

  • Later, we will convert these abstract notions of Œæ\xi, ‚Ä¢\bullet and ‚äï\oplus into concrete mathematical objects and operators.

Relative pose

The point ùêè\boldsymbol{P} can be descrbied by coordinate vectors relative to either frame {A}\{A\}, {B}\{B\} or {C}\{C\}. The frames are described by relative poses.

Coordinate frames: more complex example

  • The fixed camera observes the object and estimates the object‚Äôs pose FŒæB{^F}\xi_B.

  • The other camera is not fixed, it is attached to the robot at some constant relative pose and estimates the object‚Äôs pose CŒæB{^C}\xi_B.

  • An alternative representation of the spatial representation is a directed graph.

    • Each node represents a frame and each edge represents a relative pose.
    • Recall: we can compose relative poses, e.g.: ŒæF‚äïFŒæB=ŒæR‚äïRŒæC‚äïCŒæB,ŒæF‚äïFŒæR=ŒæR. \begin{aligned} \xi_F \oplus {^F}\xi_B &= \xi_R \oplus {^R}\xi_C \oplus {^C}\xi_B, \\ \xi_F \oplus {^F}\xi_R &= \xi_R. \end{aligned}
    • Each equation represents a loop in the graph with each side of the equation starting and ending at the same node.

Algebraic rules of relative poses

Group properties

There are just a few algebraic rules:

Œæ‚äï0=Œæ,Œæ‚äñ0=Œæ,Œæ‚äñŒæ=0,‚äñŒæ‚äïŒæ=0, \begin{aligned} \xi \oplus 0 = \xi, \quad &\xi \ominus 0 = \xi, \\ \xi \ominus \xi = 0, \quad &\ominus \xi \oplus \xi = 0, \end{aligned}

where 00 represents a zero relative pose. A pose has an inverse

‚äñXŒæY=YŒæX, \ominus {^X}\xi_Y = {^Y}\xi_X,

which is represented graphically by an arrow from {Y}\{Y\} to {X}\{X\}. Relative poses can also be composed or compounded

XŒæY‚äïYŒæZ=XŒæZ. {^X}\xi_Y \oplus {^Y}\xi_Z = {^X}\xi_Z.

A relative pose can transform a point expressed as a vector relative to one frame to a vector relative to another Xùê©=XŒæY‚Ä¢Yùê©. {^X}\bm{p} = {^X}\xi_Y \bullet {^Y}\bm{p}.

Composition is NOT commutative

Œæ1‚äïŒæ2‚â†Œæ2‚äïŒæ1, \xi_1 \oplus \xi_2 \neq \xi_2 \oplus \xi_1,

with the exception being the case where Œæ1‚äïŒæ2=0\xi_1 \oplus \xi_2 = 0.

Algebraic manipulation of poses

  • The second loop eqn. in our example says that the pose of the robot is the same as composing two relative poses:
    • from the world frame to the fixed camera,
    • from the fixed camera to the robot.

‚äñŒæF‚äïŒæF‚äïFŒæR=‚äñŒæF‚äïŒæRFŒæR=‚äñŒæF‚äïŒæR, \begin{aligned} \ominus \xi_F \oplus \xi_F \oplus {^F}\xi_R &= \ominus \xi_F \oplus \xi_R \\ {^F}\xi_R &= \ominus \xi_F \oplus \xi_R, \end{aligned}

which is the pose of the robot relative to the fixed camera, shown as the dashed line in the directed graph.

Writing these expressions by inspection

To find the pose of node XX w.r.t. node YY:

  • find a path from {Y}\{Y\} to {X}\{X\} and write down the relative poses on the edges in a left-to-right order,
  • if you traverse the edge in the direction of its arrow, precede it with the ‚äï\oplus operator, otherwise use ‚äñ\ominus.
